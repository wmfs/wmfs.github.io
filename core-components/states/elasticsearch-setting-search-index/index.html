settingSearchIndex state

Indexes an Elasticsearch document. This state is provided via the elasticsearch plugin and isn't offered by any of the other core plugins.

State config

{
  "settingSearchIndex": {
    "options": {
      "docName": "cat",
      "tags": [
        "pet",
        "cat"
      ],
      "keyPaths": {
        "name": "catDocFromParams.name"
      },
      "title": {
        "$sprintf": {
          "template": "%s (%s)",
          "inputPaths": [
            "catDocFromParams.name",
            "catDocFromParams.size"
          ]
        }
      },
      "summary": {
        "$value": {
          "path": "catDocFromParams.comment"
        }
      },
      "launches": [
        {
          "title": "Edit",
          "flowId": "catEditingFlow"
        }
      ]
    }
  }
}
Property Type Description Required
docName string Used to calculate the name of an Elasticsearch type where the document should be indexed (type names are derived as [namespace]_[docName]). Yes
keyPaths object Used to derive the id of the document to delete. Provide key/value pairs… the key will be ignored (for now), but the value should be a '.' delimited path pointing to a value on the Flobot context. These context-values will be joined with '_' to form a unique identifier for the document. Yes
title string A short label to identify a document - a good candidate for using a command (e.g. $value, $sprintf) to assemble some dynamic content from the Flobot context. Yes
summary string Provide a brief summary of the document being indexed (again consider use of $value, sprintf commands for dynamic content). No
launches array In the case where a user has searched and found the document that's to be indexed, chances are they'll want to perform an action on it (view, edit etc.) Each of the possible actions are termed a launch'. Launches are simple objects that contain the title of the action (e.g. Edit or View - but can be anything) and a flowId of any flow that can perform that action. No

autoNudge: true

This state will automatically advance the flow and doesn’t require any external updates
