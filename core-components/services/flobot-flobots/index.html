flobots service

Creates Finite State Machines based on flow definitions and an API to interact with Flobots. This service is provided via the flobot plugin and isn't offered by any of the other core plugins.

Boot config

  • This service doesn’t require any configuration to boot.

Blueprint dirs

This service can load resources defined in the following Blueprint directories:

Directory Content
/flows Flow definitions defined in JSON files

Service methods/properties

startNewFlobot

Creates and starts a new Flobot.

Parameters

  • flowId string Identifies a flow that’s been provided via a blueprint (e.g. fbotTest_cat_1_0)
  • options Object
    • options.data Object These key/values will be added to the Flobot’s context when its created.
    • options.action Object An optional string that can be used to fine-tune authorization.
    • options.userId string What userId, if any, should be associated with creating this Flobot?
    • options.onAuthorizationHook Function A function of form onAuthorizationHook (flobot, options, callback) whenever an interaction with the Flobot needs authorizing.
  • callback Function called with data relating to the freshly-created flobot

Examples

flobot.startNewFlobot(
  'fbotTest_cat_1_0',
  {
    userId: 'john.doe@flobotjs.com',
    data: {
      catName: 'Rupert'
    }
  },
  function (err, flobot) {
    // e.g:
    //   flobot.flobotId: '586d6aab8c41db1e10097db8'
    //   flobot.flowId: 'fbotTest_cat_1_0'
    //   flobot.status: 'running'
    //   etc.
  }
)

Returns undefined

updateFlobot

Updates a running Flobot.

Parameters

  • flobotId string The flobotId (as returned by startNewFlobot) of the flobot that is to be updated
  • options Object
    • options.eventId Object Explicitly defines which route the Flobot should head after completing the current state. This is optional - and only of use if Flobot wouldn’t ordinarily be able to deduce the next state automatically.
    • options.data Object These key/values will be updated on the Flobot’s context.
    • options.action Object An optional string that can be used to fine-tune authorization.
    • options.userId string What userId, if any, should be associated with this update?
    • options.onAuthorizationHook Function A function of form onAuthorizationHook (flobot, options, callback) whenever an interaction with the Flobot needs authorizing.
  • callback Function called with data relating to the post-updated flobot

Examples

flobot.updateFlobot(
  '586d6aab8c41db1e10097db8',
  {
    userId: 'john.doe@flobotjs.com',
    eventId: 'tooFull',
    data: {
      mealInfo: ['tuna', 'biscuits']
    }
  },
  function (err, flobot) {
    // e.g:
    //   flobot.flobotId: '586d6aab8c41db1e10097db8'
    //   flobot.flowId: 'fbotTest_cat_1_0'
    //   flobot.status: 'running'
    //   flobot.stateId: 'moaning
    //   etc.
  }
)

Returns undefined

getFlobot

Gets information about a running Flobot.

Parameters

  • flobotId string The flobotId (as returned by startNewFlobot) of the flobot that information is required for
  • options Object
    • options.action Object An optional string that can be used to fine-tune authorization.
    • options.userId string What userId, if any, should be associated with getting this data?
    • options.onAuthorizationHook Function A function of form onAuthorizationHook (flobot, options, callback) whenever an interaction with the Flobot needs authorizing.
  • callback Function called with the latest data relating to the flobot

Examples

flobot.getFlobot(
  '586d6aab8c41db1e10097db8',
  {
    userId: 'john.doe@flobotjs.com'
  },
  function (err, flobot) {
    // e.g:
    //   flobot.flobotId: '586d6aab8c41db1e10097db8'
    //   flobot.flowId: 'fbotTest_cat_1_0'
    //   flobot.status: 'running'
    //   flobot.stateId: 'moaning
    //   etc.
  }
)

Returns undefined

cancelFlobot

Cancels a running Flobot.

Parameters

  • flobotId string The flobotId (as returned by startNewFlobot) of the flobot that needs cancelling
  • options Object
    • options.action Object An optional string that can be used to fine-tune authorization.
    • options.userId string What userId, if any, should be associated with the cancellation of this Flobot?
    • options.onAuthorizationHook Function A function of form onAuthorizationHook (flobot, options, callback) whenever an interaction with the Flobot needs authorizing.
  • callback Function called with the information about the flobot just prior to being cancelled

Examples

flobot.cancelFlobot(
  '586d6aab8c41db1e10097db8',
  {
    userId: 'john.doe@flobotjs.com'
  },
  function (err, flobot) {
    // e.g:
    //   flobot.flobotId: '586d6aab8c41db1e10097db8'
    //   flobot.flowId: 'fbotTest_cat_1_0'
    //   flobot.status: 'running'
    //   flobot.stateId: 'moaning
    //   etc.
  }
)

Returns undefined
